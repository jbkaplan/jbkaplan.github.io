Week 4:

Arrays vs. Hashes

March 6, 2016

Well, there are a few main differences that really separate the two. Let's start first with arrays. An array is an ordered list on a computer. Each slot of the array get's assigned an index value, starting at 0, that corresponds to that variable in the array. For example, if you were to call array[0], you would get the first variable in the list. To create an array, you would have a string, float, or integer contained within brackets, as such array = [ "variable", 123, 1.234, "another variable"].

Hashes, on the other hand, also contain slots of objects, but instead of lists, they contain key-value pairs. This means each value is associated with a key, so instead of looking up a value through it's index value, like in an array, you would call hash[value], which would return the key. While arrays contain a collection of data, hashes point to the key or value. To create a new hash we can use: hash = { "key" => "value ", "key" => 123 }.

To recap, arrays and hashes are similar in that contain a collection of data and information. As well, each can be accessed using methods, and you can create methods that manipulate or change the data in an array or hash. The main thing to remember is that if you are using 2 pieces of data (key and value) you should use a hash, or if not, an array works best.
